🎯 项目一：简易21点扑克游戏（难度：⭐️⭐️） 原则：一个函数最好只做一件事。
核心目标：模拟简化版21点。玩家和电脑各随机获得2-5张牌，计算点数（J/Q/K算10，A可算1或11），接近21点但不超过的一方获胜。

阶段一：初始化。每人两张牌。
1. 创建一副牌（或设定无限牌堆的逻辑）。
2. 初始化玩家和电脑的“手牌”（空列表）。
3. 设置随机发牌的功能，这里采用有放回的抽样方法，牌堆无限厚，每位玩家都能抽取所有牌。

阶段二：遍历与分类，将手牌分为三类处理：
Ace牌 (i == 'A')：准确计数 (ace_count += 1)。✅
花牌 (J/Q/K)：通过字典正确取值为10，并累加。✅
数字牌 (2-10)：转换为整数，并累加 (total += int(i))。✅

✅ 正确的“流水线”设计思路
想象你是一个工厂的调度员，你需要安排几个工人（函数）在一条流水线上工作：
工人A（发牌员）：deal_initial_cards() -> 玩家和电脑各发两张牌。
工人B（点数计算员）：calculate_score(hand) -> 将牌堆分为三类数值，计算每人手中牌组的点数和，方便后续判断。
工人C（玩家互动员）：player_turn(player_hand) -> 负责循环询问玩家是否要牌，并调用发牌员给玩家发牌。
工人D（点数计算员）：最新发牌后立即进行计数检查，判断玩家是否爆牌。
工人E（电脑逻辑员）：computer_turn(computer_hand) -> 玩家停牌后，根据固定规则（如<17点要牌）循环为电脑发牌。
工人F （裁判员） 首先检查玩家和电脑是否爆牌（超过21点），然后比较两者点数大小决定胜负，点数相同则判为平局。如前两张牌直接达到21点时，则宣布为"Blackjack"并给予奖励判定
工人G（调度员）（主函数）：play_game() -> 组装，按顺序协调所有工人工作。
